# 语音注册命令词学习手工验证模板

本文沉淀一条已经被人工日志验证通过的命令词学习主链路，用作后续复盘自动化失败用例时的本地参考。

## 1. 正确执行顺序

命令词学习必须严格按下面阶段执行：

1. 先说唤醒词
2. 再说 `学习命令词`
3. 等第一段提示播报结束
4. 再说目标命令词
   例如：`打开风扇`
5. 等第二段提示播报结束，并确认设备进入学习态
6. 再说新的学习词
   例如：`笑逐颜开`
7. 按配置要求重复录入
8. 学习成功后回测新学习词是否能触发目标命令

禁止把下面两个阶段混在一起：

- 目标命令选择阶段
- 新学习词录入阶段

## 2. 阶段判定模板

### 2.1 唤醒成功

关键特征：

- `Wakeup`
- `wake up ready to asr mode`
- `MODE=1`
- 唤醒响应播报正常结束

### 2.2 进入学习命令词入口

关键特征：

- 识别到 `xue2 xi2 ming4 ling4 ci2`
- 出现提示播报
- 等到 `play stop`

这一步只代表进入“等待目标命令选择”的入口，不代表已经开始录入新的学习词。

### 2.3 目标命令选择成功

关键特征：

- 识别到目标命令，例如 `da3 kai1 feng1 shan4`
- 出现：
  - `study get action type: -1`
  - `Reg info: keyword[20]=da3 kai1 feng1 shan4`
- 后续出现：
  - `algo restart`
  - `voice reg type: 2`
  - `algo state init, reg status: 1`

这一步表示“要把后面的新学习词绑定到打开风扇”。

### 2.4 进入新学习词录入阶段

关键特征：

- 已出现 `reg status: 1`
- 第二段提示播报结束，看到 `play stop`
- 此时再说的内容，应该是新的学习词，不应该再说目标命令词

### 2.5 学习失败

典型特征：

- `reging process keyword: ...`
- `error cnt > N`
- `reg failed!`
- 之后进入 `reg status: 4`

如果这里的 `process keyword` 是目标命令词本身，例如：

- `reging process keyword: da kai feng shan`

那通常表示当前录入阶段说错了内容，或者新学习词被错误识别成了目标命令词。不能直接把这类现象记成“目标命令选择失败”或“模板满”。

### 2.6 学习成功

典型特征：

- `reg success!`
- `new voice.bin change`
- 之后进入 `reg status: 3`

成功后必须回测：

- 新学习词能否触发目标命令
- 默认命令是否仍可用

## 3. 用户手工通过日志的关键信号

本次人工验证里，正确链路已经证明可跑通，关键证据如下：

- `学习命令词` 后播报结束
- `打开风扇` 被识别后出现：
  - `study get action type: -1`
  - `Reg info: keyword[20]=da3 kai1 feng1 shan4`
- 之后进入：
  - `algo restart`
  - `voice reg type: 2`
  - `algo state init, reg status: 1`
- 正确录入新学习词后出现：
  - `reg success!`
  - `288 new voice.bin change`
- 回测阶段出现：
  - `Wakeup ... "keyword":"REG","intentStr":"da3 kai1 feng1 shan4"`
  - 串口发出 `A5 FA 00 81 02 00 22 FB`

这说明：

- 当前设备的命令词学习链路本身是可用的
- 不能把前面个别失败样本直接收口成“固件功能异常”

## 4. 用它怎么复判历史失败用例

后续遇到命令词学习失败时，按下面顺序复判：

1. 是否真的等到第一段提示播报结束后才说目标命令词
2. 是否真的完成了目标命令选择
3. 是否真的等到第二段提示播报结束且进入 `reg status: 1`
4. 当前失败发生在“目标命令选择阶段”还是“新学习词录入阶段”
5. `reging process keyword` 里落的到底是什么词
6. 是模板满、阶段误判、语料误识别，还是用户/脚本在学习态里说错了内容

如果失败日志满足下面模式：

- 已经有 `study get action type: -1`
- 已经有 `Reg info: keyword[20]=目标命令`
- 已经进入 `reg status: 1`
- 随后 `reging process keyword` 落成目标命令本身

则应优先收敛为：

- 录词阶段内容错误
- 新学习词被错误识别为目标命令
- 或自动化阶段切分不准确

不应直接收口为：

- 目标命令选择失败
- 固件不支持学习命令词
- 模板满

## 5. 当前对历史失败项的收敛原则

基于该模板，之前命令词学习失败项需要按以下口径收敛：

- 若失败发生在 `reg status: 1` 之后，先检查它是不是已经进入“新学习词录入阶段”
- 若 `process keyword` 落成目标命令词，优先记为“录词阶段失败”，不是“入口失败”
- 若没有任何模板满/容量满日志，不得擅自归因为“模板满”
- 只有在阶段证据完整、且新学习词本身也明确正确时，才可以把失败记到固件逻辑上
