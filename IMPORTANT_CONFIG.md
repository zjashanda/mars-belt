# Important Config

## Root Base Config

- Root file: `deviceInfo_generated.json`
- Purpose: persistent global `ttsConfig` and audio-card base config for this skill
- This file must be carried together when moving/copying/backing up `mars-belt`

## Behavior

- New suites generated by `generate-suite` or profile-based suite generation will inherit this `ttsConfig`.
- New suites will also inherit `deviceListInfo.audioCard` as the default playback target.
- `deviceKey` is the stable ListenAI audio device key. When it is usable, playback binds to that card first.
- If the configured card is unusable, playback may fall back to the default sound card only when there is at most one active `ListenAI Audio` render device.
- If multiple `ListenAI Audio` render devices are active, default fallback is blocked and the test runner must stop with an explicit error so testers can choose a specific card.
- `package-weekly` will also prefer this root config as TTS fallback.
- If this file is missing, voice-related suites may stop at audio preparation when local MP3 files are incomplete.

## Root Memory And Result

- Root `memory/` and root `result/backups/` are created by the internal `backup-state` flow.
- They are backup artifacts, not firmware test delivery directories.
